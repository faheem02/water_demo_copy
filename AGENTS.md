# Water Supply Management System — AGENTS.md

## Stack
- **PHP 8+** vanilla (no framework), **MySQL** via `mysqli_*`, Bootstrap 5.3.3, jQuery 3.7.1, Font Awesome 6.5.2
- Runs on **XAMPP** (localhost). DB: `water_supply_system`, user: `root`, pass: empty (`includes/db.php:3-6`)

## Setup
1. Create DB `water_supply_system` and import `database/schema.sql` (21 tables + default admin + expense categories)
2. Or hit `http://localhost/water_demo_copy/database/database.php` once
3. Login: `admin` / `admin123` at `login.php`

## Architecture
- **Entrypoint**: `index.php` (dashboard). Auth check: `$_SESSION['admin_logged_in']`
- **Root pages**: `login.php`, `logout.php`, `index.php`
- **Feature pages**: all in `pages/` (23 files — customers, deliveries, stock, payments, etc.)
- **Includes**: `includes/db.php` (session + DB), `includes/header.php` (layout + sidebar + mobile menu JS), `includes/footer.php` (jQuery + Bootstrap JS), `includes/txt.php` (site config strings)
- `includes/sidebar.php` is a stub — sidebar is rendered inside `header.php`

## Site config (`includes/txt.php`)
Defines `$software_name`, `$company_name`, `$owner_name`, `$owner_address`, `$owner_phone`, `$footer_text`. Included by `db.php` and `header.php`.

## Quirks
- **No build tools, tests, linters, CI, or README**
- Auth uses **plain-text password comparison** (`login.php:14`)
- SQL uses string interpolation with `mysqli_real_escape_string` — no prepared statements
- Paths use auto-detected `$base_url` in header — `'../'` when in `pages/`, `''` otherwise
- All CSS and JS on page scripts is **inline** in PHP files (no external modules)
- `assets/js/script.js` is empty (unused)
- `error_log` files sit at root and in `pages/` — generated by runtime errors

## Testing
- No test framework exists
- Manual smoke test: log in, visit each `pages/` endpoint, verify CRUD operations

## Key schema notes
- **Routes** (`routes` table): `route_name`, `block`, `area`, `salesman` (no `description`)
- **Customers** (`customers` table): has its own `block`, `area`, `salesman` fields, pre-filled from selected route on add/edit (`pages/customers.php`)
- **Deliveries** (`pages/deliveries.php`): when a customer is selected, shows route name, block, area, salesman inline
